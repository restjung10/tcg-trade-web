# 프로젝트 현황 (최종 갱신: 2026-07-20)

> 이 섹션은 다른 환경(claude.ai/code 웹, 데스크톱 앱, CLI 등)에서 새 세션을

> `~/.claude` 메모리는 이 저장소를 클론한 다른 환경에서는 접근 불가능하므로,

## 개요
"TCGinside"(GitHub: restjung10/tcg-trade-web) — TCG 카드 거래 커뮤니티.
개인 취미 프로젝트, 예산 최소화 우선, 사업자등록 없이 개인 운영자로 운영.
사기 방지가 핵심 목적(과거 노출되던 태그라인은 삭제했지만 목적 자체는 유지).

**스택**: Next.js 16(App Router, TS, Tailwind v4) + Supabase(Postgres/Auth/Storage/Realtime,
무료 티어) + 카카오 로그인 + Sightengine(AI 생성 이미지 검증) + Web Push(VAPID) +
Vercel Hobby 배포(서울 icn1 리전, `https://tcginside.vercel.app`).

## 완료된 기능
- 카카오 로그인/온보딩(닉네임+약관동의)/마이페이지/회원탈퇴(소프트삭제)/관리자 차단
- 게시판 CRUD(판매/구매 × 완덱/낱장), 검색(제목+본문/글쓴이), 조회수, 끌어올리기(1일1회)
- 이미지 업로드 + Sightengine AI생성이미지 검증 + 워터마크 + 압축(1600px/q82)
- 1:1 채팅(Realtime) + 웹 푸시 알림 + 채팅방 상단 게시글 배너
- 계좌 인증(관리자 승인 게이트, AES-256-GCM 암호화) → 글쓰기/채팅 선행 조건
- 거래 절차(계좌전송→입금확인→송장전송→수령확인, 자동완료) + 작성자 직접 완료
  전환(채팅 1건 이상 필요, 시세조작 방지) + 거래 취소 + 이중거래 방지
- 신고/정지/차단, /admin/users 전체회원 검색+정지+차단, /admin/reports, /admin/bank-accounts
- 개인정보처리방침/이용약관/통신판매중개자 고지(전자상거래법 20조), 전 페이지 Footer
- PWA manifest, 라이트/다크 모드 수동 토글
- PII 컬럼 단위 잠금(GRANT 기반), rate limiting, 각종 RLS/트리거 보안 강화
- (2026-07-20) 게시글/이미지 삭제 시 Storage 파일도 실제 삭제(pending 원본 leak,
  final 버킷 orphan 문제 수정) — 과거 누적된 leak 파일도 정리 완료

## 알려진 미해결 이슈 (우선순위순, 실사용자 공개 전 검토 필요)

**아직 구현 안 한 것 (제안만 하고 승인 대기 중):**
1. **Sightengine 쿼터 소진 시 fail-closed** — `lib/image/process.ts`가 API 실패(월
   2,000 operation, genai 5op/장 소모 = 실질 월 400장 한도) 시 예외를 던지고
   `createPost`가 이를 판매글 작성 완전 차단으로 처리함(`lib/actions/posts.ts` 93-98행).
   제안: fail-open으로 바꿔 검증 실패 시 `post_images.verification_status`를
   별도 상태(예: `unverified`)로 남기고 통과시키기.
2. **계좌 인증이 실명 대조 없는 자기신고** — `/admin/bank-accounts`는 사용자가 입력한
   텍스트를 관리자가 눈으로 보고 승인 버튼 누르는 것뿐, 실제 은행 시스템과 대조 안 함.
   무료 완화책: 안내 문구로 "형식 확인 수준"임을 명시. 유료 완전 해결책: 예금주
   실명조회 API(건당 수십 원).
3. **밴/차단이 카카오 계정 재생성으로 우회 가능** — `blocked_kakao_users`는
   kakao_user_id 기준인데 카카오 계정은 무한 재생성 가능. 게다가 `lib/crypto.ts`의
   계좌번호 암호화가 매번 랜덤 IV라 같은 계좌번호도 암호문이 매번 달라져서, 이미
   차단된 계좌의 재사용을 DB에서 대조할 방법이 구조적으로 없음. 제안: 계좌번호에
   HMAC-SHA256 블라인드 인덱스(결정적 해시)를 추가해 재사용 탐지 + (선택) SMS
   본인인증(알리고/네이버클라우드 SENS — 사업자등록 불필요, 개인 가입 가능) 도입.
4. **게시글 삭제 시 채팅 기록도 cascade로 통째로 삭제됨** — `chat_rooms.post_id`가
   `on delete cascade`라 작성자가 글을 지우면 관련 채팅(분쟁 시 증거가 될 수 있는
   기록)까지 전부 사라짐. **사용자에게 아직 보고 안 한 상태** — 다음 세션에서
   우선 보고할 것. 해결 방향(미확정): 채팅이 있는 글은 삭제 대신 비공개/보존
   처리하거나, cascade를 끊고 별도 정책 적용.
5. **신고/계좌인증 신청에 대해 관리자 알림 없음** — 운영자가 능동적으로
   `/admin/reports`, `/admin/bank-accounts`를 확인해야만 알 수 있음. **아직
   보고 안 함.**
6. **에러 모니터링(Sentry 등) 부재** — 프로덕션 에러를 사용자 신고 없이는 알 방법이
   없음. **아직 보고 안 함.**
7. **Supabase 무료 티어는 백업/PITR 미포함** — 실수로 데이터 삭제 시 복구 불가.
   수동 `supabase db dump` 등으로 주기적 백업 필요. **아직 보고 안 함.**

**구조적으로 완전 해결이 어려운 것 (인지만 하고 있으면 되는 것):**
8. 통신판매업 신고 의무 여부 — 거래규모/수수료수취에 달린 법률 판단, 코드로 해결 불가.
   거래량이 늘면 재검토 필요.
9. 운영자 1인 병목(신고/승인/문의 전부 혼자 처리) — 완전 자동화 어려움.
10. 에스크로 부재 — 실제 결제는 플랫폼 밖에서 이뤄져 분쟁 시 회사가 보상 불가.
    전자금융업 라이선스 필요해 지금 규모에서 불가능. 이용약관에 면책 조항은 반영됨.

**비용 관련 (모니터링만 하면 되는 것):**
11. Supabase Storage egress — 이미지가 Vercel을 안 거치고 Supabase에서 직접
    서빙되어 여기가 진짜 병목. leak은 고쳤으니 실사용자 늘면 대시보드 Usage 탭
    주기 확인 권장. Sightengine 유료 전환 시 Starter($29/월, 월 2,000장)가
    자연스러운 다음 단계.
